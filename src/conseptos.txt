Que es una Api
una api es un contrato de especificaciones(definiciones,protocolos)
se define para que dos o mas componentes tecnológicos se comuniquen entre si,
una api encapsula toda su lógica que implementa para cumplir con estos contratos
Api estilo arquitectónico:
RESTful-> implentan el estilo arquitectónico REST -> formato principal (json)
SOAP-> basadas en XML(esquemas muy rigidos) es muy complicado cambiar el formato el cliente debe tener un esquema muy rígido
GRPC-> tipo de api de comunicación basado en http (2) versión mas avanzada (se dice que es mas eficiente para comunicaciones internas)
ASINCRONAS-> apis que se manejan con mensajería o colas no es un api web (es un api que se basa en mensajeria donde necesito un
desacoplamiento esos componentes ósea que no hay dependencia en la tecnologia
GRApHQL->lenguaje d consultas para apis (el cliente define lo que nesecita)
